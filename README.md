# Аналізатор веб-посилань (Web Crawler & Link Scanner)

## Про продукт

Аналізатор веб-посилань — це потужний та оптимізований інструмент для глубокого сканування веб-сайтів. Додаток дозволяє автоматично обходити сторінки заданого ресурсу, виявляти структуру посилань та ефективно знаходити «биті» (непрацюючі) лінки.

Завдяки використанню гібридного підходу до сканування (поєднання швидкого асинхронного HTTP-клієнта `aiohttp` та повноцінного рушія `Playwright` для рендерингу JavaScript), система здатна обробляти як класичні статичні сайти, так і сучасні Single Page Applications (SPA), створені на базі React, Vue, Angular тощо.

## Проблематика, яку вирішує застосунок

1. **Пошук непрацюючих посилань (Broken Links):** Биті посилання негативно впливають на користувацький досвід та SEO-рейтинг сайту. Ручна перевірка тисяч сторінок є неможливою. Цей застосунок сканує та автоматично формує звіт про їх наявність.
2. **Підтримка сучасних веб-технологій:** Більшість базових краулерів не здатні розпізнавати контент, який генерується динамічно за допомогою JavaScript. Цей інструмент використовує браузерний рушій для коректного рендерингу таких сторінок, що гарантує точність зібраних даних.
3. **Оптимізація часу та ресурсів:** Використання гібридного підходу дозволяє сканувати сотні сторінок за лічені секунди, активуючи ресурсоємний рендеринг лише там, де це дійсно необхідно (наприклад, при блокуванні анти-бот системами або відсутності базового HTML-контенту).
4. **Масштабованість та фонова обробка:** Завдяки використанню черг завдань (Celery + Redis), система може паралельно обробляти великі обсяги даних у фоновому режимі, не блокуючи основний веб-інтерфейс.

## Технологічний стек

- **Backend:** Python, Django
- **Асинхронні задачі:** Celery, Redis
- **Сканування:** aiohttp, BeautifulSoup4, Playwright
- **Інфраструктура:** Docker, Docker Compose

## Інструкція з розгортання

Для успішного запуску проєкту на вашому локальному комп'ютері повинен бути встановлений **Docker** та **Docker Compose**.

### 1. Клонування репозиторію

Склонуйте проєкт на свій комп'ютер та перейдіть до директорії проєкту:

```bash
git clone <URL_репозиторію>
cd Web-crawler
```

### 2. Запуск контейнерів

Побудуйте та запустіть контейнери у фоновому режимі за допомогою Docker Compose:

```bash
docker-compose up -d --build
```

Ця команда автоматично завантажить необхідні образи, встановить залежності Python (включно з браузерами Playwright), налаштує Redis та запустить веб-сервер разом із Celery-воркерами.

### 3. Застосування міграцій бази даних

Після успішного запуску контейнерів виконайте міграції для створення необхідних таблиць у базі даних SQLite (використовується за замовчуванням):

```bash
docker-compose exec web python manage.py migrate
```

### 4. Створення суперкористувача (опціонально)

Для доступу до панелі адміністратора створіть обліковий запис адміністратора:

```bash
docker-compose exec web python manage.py createsuperuser
```

Дотримуйтесь інструкцій на екрані для введення логіна, email та пароля.

### 5. Доступ до застосунку

Після виконання всіх кроків застосунок буде доступний у вашому браузері за адресою:

**http://localhost:8000**

Система повністю готова до роботи. Ви можете зареєструватись, задати цільовий URL та глибину сканування для початку аналізу вашого першого веб-ресурсу.
